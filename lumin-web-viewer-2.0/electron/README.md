# Electron App - Environment Configuration

This directory contains the Electron desktop application for Lumin PDF.

## 🔧 Environment Configuration

The app uses a **branch-based configuration system** to connect to different backend environments.

### Development Mode
- No configuration file needed
- Automatically uses `localhost:3000`
- Run with: `pnpm run electron:dev`

### Production Mode
- Requires `electron-config.json` to specify the branch
- Generated automatically during build
- Branch determines which backend URLs to use

## 📦 Building for Different Environments

### Quick Start

```bash
# Development (localhost)
pnpm run electron:dev

# Production build (default: develop branch)
pnpm run electron:dist

# Production build for specific branch
pnpm run generate-electron-config -- --branch=production
pnpm run electron:dist:mac

# Or use environment variable
LUMIN_BRANCH=staging pnpm run electron:dist:win
```

## ⚙️ How It Works

### 1. Build Time Configuration
The build process uses **two layers of configuration**:

#### a) Electron Builder Config (`electron-builder.config.js`)
Dynamically sets the app metadata based on `LUMIN_BRANCH`:
- **productName**: e.g., "Lumin for Desktop (Viewer Staging)"
- **appId**: e.g., "com.lumin.pdf.viewer-staging"
- File associations, icons, and platform-specific settings

```bash
# Example: Building for viewer-staging
LUMIN_BRANCH=viewer-staging pnpm run electron:dist:mac
```

Output:
- Product Name: "Lumin for Desktop (Viewer Staging)"
- App ID: "com.lumin.pdf.viewer-staging"

#### b) Runtime Config (`electron/electron-config.json`)
Generated by `generate-electron-config` script:
```bash
pnpm run generate-electron-config -- --branch=production
```

Creates:
```json
{
  "branch": "production",
  "generatedAt": "2025-10-02T12:00:00.000Z"
}
```

### 2. Runtime
`EnvironmentService` reads the bundled config and loads the appropriate settings from `settings/<namespace>/<branch>.json`

### 3. Result
Each build is uniquely configured:
- **App Name**: Different for each environment (visible in menu bar, dock, task manager)
- **App ID**: Unique identifier allowing multiple versions to be installed simultaneously
- **Backend URLs**: `LUMIN_BASEURL`, `LUMIN_GOOGLE_PICKER_CLIENTID`, `LUMIN_AUTH_SERVICE_URL`

## 🔍 Troubleshooting

**Error: "electron-config.json not found"**
- Run `pnpm run generate-electron-config -- --branch=<branch-name>` before building
- Or set `LUMIN_BRANCH` environment variable

**Wrong backend URL in production**
- Check which branch was used during build
- Rebuild with correct branch flag

## 📝 Notes

- Development mode never creates `electron-config.json`
- Production builds require the config file
- The config file is bundled into the app at build time
- Branch determines all backend URLs and OAuth settings
- **Dynamic Configuration**: `electron-builder.config.js` uses `LUMIN_BRANCH` to set product name and app ID
- **Multiple Installations**: Different app IDs allow installing multiple environments side-by-side
- **Consistent Branding**: Each environment has a distinct product name visible to users

## 🎯 Advanced Configuration

To customize the configuration for a specific branch, edit `electron-builder.config.js`:

```javascript
const branchConfig = {
  'your-branch': {
    productName: 'Your Custom Name',
    appId: 'com.lumin.pdf.your-branch',
    // Add more custom settings if needed
  },
  // ... other branches
};
```

Available customizations:
- `productName`: Application display name
- `appId`: Unique identifier (format: `com.lumin.pdf.<branch>`)
- `icon`: Custom icon path per environment
- `publish`: Different publish settings per environment
